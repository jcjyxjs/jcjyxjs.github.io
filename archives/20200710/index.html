2020年7月10日-记录网站重新搬迁

2020-07-11 jcjyxjs

前言

一开始我是使用阿里云香港轻量，也是 Typecho + VOID 的形式，搭建的博客，那时候的域名是 jcjyxjs.me ，是 Github 学生包里送的。3个月之后发现阿里云香港不走 CN2 了，加上晚上一直检测到无法访问。我就想转到国内，突然就想到还有这么一个备案域名。

于是我使用 Maverick 这个程序搭建的静态博客，因为正好这个主题的作者也在推 Maverick ，类似于 Hexo ， Hugo 等。

使用腾讯云对象存储（COS）+内容分发网络（CDN）的形式。COS 开启静态网站支持。就这样持续了有个大半年。我突然意识到， Maverick 主题有点少，而且没有暗黑模式自适应（没说Maverick不好），我又太菜，只会一个暗黑模式自适应。最近就看到腾讯云新用户 1H2G1M 99/年 :%(酷安_doge呵斥) 果断入手。

7月10号中午入手，装完环境，配置好CDN，我就开冲了。

首先

  1. 我需要把所有文章一篇一篇导入，我不会批量导入，也怕批量导入出现各种错误整的到时候还是要一篇一篇手动导入。
  2. 需要导入 jcjyxjs.me 这个博客的评论，以及使用 Maverick 是在 Leancloud 的评论。这个也只能一条一条导入然后修改信息。

我当时的数据库备份和根目录备份是很早的，基本没什么文章，还不如重新安装重新弄过。文章一篇一篇复制粘贴然后发布没什么好说的。主要是评论。

原来的评论，我只能用评论者的 Name，Mail，[Link] 在我新的站里面重新评论，然后进入 MySQL 一条一条改。更要命的是， Leancloud 后台评论时间是具体的 XXXX年XX月XX日 XX:XX 。

而 MySQL 是10位时间戳，就是说我得把具体时间转换成时间戳。

由于是重装的，主题啥的得重新配置。服务器只有1M，又追求速度，只能配置静动分离。（视频，图片，音乐，js，css等）全部托管在COS，减少源站压力，服务器安装 BBR ，并不使用 Redis 等进行缓存，之前看其他博客说会把密码缓存了 :%(酷安_流汗滑稽) 然后就是把一些文章的标题图片全部加回来。

最后，我给主题加了两组表情包，分别是2233娘和酷安（部分） :*(2233_大笑) 如果站点有任何问题也请直接留言告诉我（开启了评论审核），谢谢~ :%(酷安_py交易)
